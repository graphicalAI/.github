# Руководство по разработке для команды graphicalAI

Рады, что вы с нами! Пожалуйста, следуйте правилам ниже для сохранения чистоты кодовой базы и успешной отчетности по гранту ФСИ.

## 🌿 Правила ветвления и Git-процесс

Мы используем упрощенный Git Flow:
1.  **`main`** — только для стабильного production-кода. Коммитить напрямую строго запрещено.
2.  **`develop`** — основная ветка разработки. Все новые фичи вливаются сюда через Pull Request.
3.  **Именование веток**:
    *   `feature/короткое-название` — для нового функционала.
    *   `bugfix/короткое-название` — для исправления ошибок.
    *   `hotfix/короткое-название` — для срочных исправлений в `main`.

## 📝 Правила оформления коммитов

Все коммиты в репозитории организации должны оформляться строго по шаблону:
```text
#{номер задачи} {статус[WIP/Done/Fix]} Краткое описание
```

### Допустимые статусы:
*   `WIP` (Work In Progress) — задача в процессе разработки, код еще не завершен.
*   `Done` — внедрение фичи завершено, задача готова к проверке.
*   `Fix` — исправление багов, баг-фиксы или мелкие правки в коде.

### Примеры правильных сообщений коммитов:
*   `#42 WIP Реализация коннектора для сокетов`
*   `#42 Done Добавлена отправка сетевых состояний графа`
*   `#15 Fix Исправлено падение при удалении связей`

## 🔄 Процесс работы над задачей

1.  Выберите задачу на доске **GraphNet Project** (или получите от Георгия).
2.  Перенесите карточку задачи в статус `In Progress`.
3.  Создайте ветку от `develop`.
4.  Напишите код и тесты. Локально запустите проверку.
5.  Создайте Pull Request (PR) из вашей ветки в ветку `develop`.
6.  Переместите задачу в статус `At Review` и отправьте ссылку на PR Георгию (`jezvgg`).
7.  После успешного ревью изменения будут влиты в `develop`.

## 📜 Важно для отчетности по гранту ФСИ
Если вы работаете по договору ГПХ с ООО в рамках гранта ФСИ, при постановке задач и описании PR обязательно соотносите формулировки с этапами Календарного плана гранта.
